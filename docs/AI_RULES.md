# AI 协作规则（MUST / SHOULD / MAY）

## 1) MUST（必须遵守）
- 不制造压力：不输出“必须修/警告/惩罚性”措辞。
- 工具定位：以“预览与交付”为核心，不以“诊断找错”为核心。
- 展示优先：SERP/分享卡片预览为主，诊断项不作为主视图。
- 文档分区：指导性文档放 `docs/`，临时/协作放 `notes/`（见 `docs/DOCS_RULES.md`）。
- 真实一致性：预览与测量逻辑必须一致，避免误报。

## 2) SHOULD（强烈建议）
- 极简实用：功能数量少但可用，信息层级清晰。
- 文案短句：仅陈述事实与展示效果，避免命令式语气。
- 默认低噪声：只展示最必要的信息，其余折叠或隐藏。
- 组件层级：预览 > 摘要/导出 > 元信息快照。

## 3) MAY（可自由发挥）
- 按钮具体名称（在不违背 MUST 的前提下）。
- 视觉细节（圆角、阴影、间距微调）。
- 动效与小交互（不影响加载与可读性）。
