# 设计规范（方向稿）

## 1. 视觉目标
- 极简、高级、截图好看
- 信息层级清晰，诊断提示一眼可见
- 预览组件高度拟真（Google/分享卡片）

## 1.1 高级感基准（可执行标准）
- 克制：少色彩、少装饰、少分隔线
- 排版：标题明确、层级拉开、行距舒适
- 比例：留白 > 填充，卡片与文字比例稳定
- 一致性：间距、圆角、阴影统一
- **材质（MVP 修正）**：放弃毛玻璃/Bkd Blur（性能考量），采用纯色背景 + 微底边框 + 多重阴影来体现层次。

## 2. 设计原则
- 去杂：避免多余说明、提示和“未来功能”露出
- 高对比但不刺眼：用于展示重点信息
- 统一卡片体系：预览/诊断/按钮都在同一视觉语言
- **精密感**：数字必须等宽对齐，动画必须快速且有秩序。

## 3. 色彩系统（建议）
- 背景（暖白）：#F7F6F2 / #FAF9F6（二选一）
- 主文本：#141414
- 次文本：#6B6B6B
- 边框：#E8E6DF
- 卡片底色：#FFFFFF
- 强调色（低饱和蓝灰）：#2F5BFF
- 警告色（低饱和橙）：#D98A3A
- 严重色（低饱和红）：#C5534B

## 4. 字体与排版（核心升级）
- 中文主字体：Source Han Sans SC / Noto Sans SC
- 英文辅助：Sora / Space Grotesk
- **精密排版**：
  - **数值**：强制使用 `tabular-nums`（等宽数字），确保列表和计数器不抖动。
  - **大标题**：`letter-spacing: -0.03em`（紧凑有力）。
  - **标签**：`letter-spacing: 0.05em` + Uppercase（精致秩序）。

## 5. 排版与间距
- 基础间距：8px 体系（8/16/24/32）
- 卡片圆角：12px（统一）
- 线条：1px 细描边
- 行高：1.5 - 1.7

## 6. 阴影与层次（高级感来源）
- **多重阴影 (Layered Shadows)**：使用多层阴影代替单层简单阴影。
  - 例：`box-shadow: 0 1px 2px rgba(0,0,0,0.04), 0 4px 12px rgba(0,0,0,0.02)`
- 模态/浮层：略重于卡片，保持克制。

## 7. MVP UI 规格（结构 + 尺寸）
### 弹窗 Popup
- 宽度：360px（允许 340-380 适配）
- 内边距：16px；模块间距 12px；卡片内边距 12px
- 顶部栏：Logo +「SEO 预览」+ 域名；右侧状态徽标（OK/问题数）
- 操作区：主按钮「完整预览」，次按钮「刷新」
- 预览区：SERP 卡片 + 分享卡片缩略卡（优先 OG；无 OG 则回退分享元信息）
- 诊断区：Top 3 问题，小卡片列表，含严重度标签与一句话结论
- 诊断项结构：严重度标签 + 规则名 + 一句话结论（不展开建议）

### 完整预览页
- 布局：左预览 / 右诊断，比例 3:2（窄屏单列）
- 顶部栏：标题 + 域名 + 状态摘要；操作按钮固定在右侧
- 预览区：SERP（桌面/移动切换）+ 分享卡片大卡片
- 诊断区：按严重度分组，展开建议不堆在列表里
- 诊断项结构：严重度标签 + 规则名 + 一句话结论 +「查看建议」展开

## 8. MVP UI 规格（文本 + 展示）
- 字号建议：标题 16/18px，正文 13/14px，次文本 12px，标签 11px；行高 1.5
- SERP 预览：标题单行、描述两行；像素宽度提示仅在完整预览显示
- 分享卡片：1.91:1 主图；标题 1 行、描述 2 行；无图显示占位块 +「无预览图」
- 严重度样式：Critical=红、Warning=橙、Info=蓝灰；仅用于标签/边框
- 状态页：加载/空数据/不可扫描均用同样卡片样式，不使用插画
- 异常落位：异常状态卡优先放在预览区位置；诊断区显示“暂无问题/无法生成诊断”

## 9. 导出 PNG 版式（MVP 规格）
- 导出内容：SERP（当前设备）+ 分享卡片 + 问题摘要（Top 3）
- 版式：纵向堆叠，整体外边距 24px，区块间距 16px
- 尺寸：导出宽度 1200px，自动高度；生成时按 2x 像素密度输出
- 背景：与页面背景一致（暖白）；不加品牌水印
- 文字与卡片样式：与完整预览页一致

## 10. 组件建议
- 预览卡片：
  - SERP 桌面/移动双卡
  - 分享卡片
- 诊断提示：
  - 使用小标签 + 简短句
  - 仅突出关键问题
- 按钮：
  - 单一主按钮（减少选择成本）

## 11. 内容策略
- 文案短、准、专业
- 术语尽量清晰，避免 SEO 行话堆叠

## 12. 输出要求
- UI 支持一键截图导出（若实现导出功能）
- 在社媒截图传播时信息完整
