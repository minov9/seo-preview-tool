# 产品需求文档（PRD）

## 1. 产品概述
一款专注于“当前页面即时预览 + 规范检查”的 SEO 预览工具，先做 Chrome 插件（MVP），后续可拓展到网页端批量检查。强调极简、专业、可截图传播的 UI。
术语：分享卡片=链接预览卡片（基于 OG 等元数据生成）。

## 2. 核心卖点
- 当前页面即时预览：无需复制/粘贴 HTML，打开页面即检查。
- 高还原预览：SERP（桌面/移动）+ 分享卡片预览。
- 诊断直观：缺失/过长/格式问题快速提示。
- 极简高级 UI：适合社媒截图展示。

## 3. 目标与成功标准
- 目标：用最小成本上线并验证需求，形成可收费的 MVP。
- 成功标准：
  - 用户可在 30 秒内完成一次检查并理解问题点。
  - 预览效果“像真实平台”，适合截图传播。
  - 付费意愿来自“更快交付/更少出错”。

## 4. 目标用户
- 主用户：独立站长、内容运营、SEO 从业者、增长/市场人员。
- 次用户：代理/外包交付团队（需要截图/摘要交付）。
- 开发者：仅作为辅助使用者，不是主目标。
- 市场侧重点：前期以国内用户为主，后续兼顾海外用户。

## 5. MVP 范围（已定）
- 形态：Chrome 插件优先。
- 语言：中文为主，预留 i18n 架构。
- 支付：仅人民币（支付宝），不展示美元/估算价。
- 视觉：极简、高级、可截图传播。
- Free：不做次数限制，仅做功能限制。

## 6. MVP 功能清单（建议优先级）
### P0（必须）
- 读取当前页面的 Title / Meta Description / OG / 分享元数据。
- SERP 预览（桌面/移动）。
- 分享卡片预览（OG 等）。
- 基础诊断：缺失/过长/过短/图片尺寸不合规等。
- 像素宽度截断提示（优先于纯字符计数）。
- 关键风险检查：canonical/noindex/H1。

### P1（Pro，包含）
- 一键导出：导出 PNG + 文本摘要复制（用于交付/复盘）。
- 规则提示增强：一致性与重复性提示（Pro）。

### 高级诊断规则（Pro）
- Title/Description 像素宽度截断风险（更精细阈值）。
- Title 与 H1 高度重复提示（避免机械复制）。
- Title/Description 与分享文案不一致提示。
- `og:image` 尺寸与比例不合规提示。
- `twitter:card` 缺失或类型不匹配提示。

### P2（后续）
- 批量 URL 检查（Web 端）。
- 历史对比 / 变更提醒。
- 团队协作与项目管理。

## 6. MVP 交互结构（建议）
- 主入口：点击插件图标即自动扫描并展示结果。
- 核心模块：
  - 概览区：一眼看到主要问题数量与等级。
  - SERP 预览：桌面/移动切换。
  - 分享卡片预览：卡片样式展示。
  - 诊断列表：缺失/长度/图片尺寸等问题点。
- 核心动作（按钮层级）：
  - 刷新/重新扫描
  - 复制/导出（如启用导出功能）
  - 设备切换（桌面/移动）

## 10. 插件弹窗布局（已定）
- 顶部栏：Logo + 页面标题（截断）+ 域名（小字）+ 状态徽标（OK/问题数）。
- 操作区：次按钮“刷新/重新扫描”，主按钮“打开完整预览”。
- 预览区：
  - SERP 预览（单卡片，桌面/移动切换）。
  - 分享卡片预览缩略卡（小图+标题），完整卡在完整预览页。
- 诊断区：
  - 仅展示 Top 3 问题（按严重度排序）。
  - “查看全部”跳转完整预览页。
- 交互规则：
  - 打开弹窗自动扫描，无“开始”按钮。
  - 动态页面可手动刷新。
  - 导出能力仅在完整预览页（Pro）。

## 11. 异常状态（弹窗）
- 内置页面（chrome://、Chrome Web Store）：提示无法扫描。
- 元信息缺失：明确提示缺失项。
- 图片不可访问：提示跨域或加载失败。

## 12. 完整预览页布局（MVP，极简）
- 布局：宽屏双栏（左预览、右诊断），窄屏单列。
- 顶部栏：标题 + 域名 + 状态摘要；操作按钮仅保留“刷新/导出 PNG/复制摘要”。
- 预览区（左）：
  - SERP 预览（桌面/移动切换）
  - 分享卡片大卡片预览
- 诊断区（右）：
  - 问题总览 + 按严重度分组（Critical/Warning/Info）
  - 每条仅一句话结论（可展开短建议）
- 详情区（可折叠）：元信息明细（Title/Description/OG/分享/Canonical/Robots/H1）。
- 导出：由导出按钮生成 PNG（不要求用户手动截图）。
- 导出 PNG 版式（Pro）：SERP（当前设备）+ 分享卡片 + 问题摘要（Top 3）。
- 视觉：背景使用极浅灰/米白（非纯白）；卡片纯白 + 轻阴影；标签淡色块 + 细描边。
- 不使用水印，保持克制高级。

## 7. 用户买单点（方向）
- 可交付：导出 PNG + 文本摘要，作为对外沟通与交付物。
- 更严格诊断：更细规则、更高质量提示（Pro）。
- 效率提升：批量检查与清单优先级（后续 Web 端）。

## 8. 非目标（MVP 不做）
- 深度爬虫与复杂 SEO 评分体系。
- GA/GSC 等深度数据集成。
- 多语言完整覆盖（先中文 + 架构）。
- 复杂 AI 文案生成。
